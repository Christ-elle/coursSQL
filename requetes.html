Les conditions: le mot clef WHERE

Nous cherchons avec une condition une partie des données.

Affiche-moi le nom et le prénom de la table des employés à condition que les employés fassent parti du service informatique

  • SELECT nom, prenom FROM employes WHERE service='informatique';
  • Ou l'employé dont le prénom est Thomas: SELECT nom FROM employes WHERE prenom='thomas';

Le mot clef BETWEEN

Afficher les employés ayant été recrutés entre 2006 et 2010

  • SELECT * FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND '2010-12-31';
  • SELECT nom, prenom date_embauche FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND '2010-12-31';
  • Dans une BDD la date est au format états-uniens ANNEE-MOIS-JOUR // 2021-02-23
  • Une requête qui donne la date du jour SELECT CURDATE()
  • Entre une date et aujourd'hui SELECT * FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND CURDATE();

Le mot clef LIKE

Afficher les employés dont le prénom commence par S; ce qui sera très utile dans un moteur de recherche

Avec la présence des signes % avant et après la lettre ou le mot recherché, nous demandons au système de trouver des enregistrements contenant l'expression recherchée

  • SELECT * FROM employes WHERE prenom LIKE 's%';
  • Pour selectionner tous les prénom qu ise trminent par s SELECT * FROM employes WHERE prenom LIKE '%s';
  • Pour selectionner tous les prénom qui comportent la lettre s SELECT * FROM employes WHERE prenom LIKE '%s%';

L'opérateur

  • Tous les employés qui ne sont pas au service informatique SELECT * FROM employes WHERE service !='informatique';

L'opérateur suppérieur >

  • Tous les employés gagants plus de 3000 Euros SELECT nom, prenom, service, salaire FROM employes WHERE salaire > 3000;

Le classement avec ORDER BY

  • Les employés triés par nom de famille et par ordre alphabétique SELECT prenom, nom FROM employes ORDER BY nom;
  • Avec ou sans ASC la requête est la même par défault SQL tri du plus petit au plus grand
  • ASC pour ascendant et DESC pour descendant
  • ......
  • Trier sur plusieurs colonnes SELECT nom, prenom, service, salaire FROM employes ORDER BY salaire ASC, nom ASC;

Limiter les résultats avec le mot clef LIMIT

  • Afficher les employés en les classant par salaire (du + grand au + petit) et en les affichant par 3SELECT nom, prenom, service, salaire FROM employes ORDER BY salaire DESC LIMIT 0,3;
  • Le premier chiffre précise l'enregistrement de départ (0 définit le 1er enregistrement en informatique, le 2eme chiffre nous indique le nombre de résultats que l'on souhaite obtenir
  • Autre ex: SELECT nom, prenom, service, salaire FROM employes ORDER BY salaire DESC LIMIT 6,3;
  • Le mot clef LIMIT est utile pour faire une pagination web

Réaliser des calculs dans nos requêtes

  • Afficher la liste des employés avec leurs salaire annuel SELECT nom, prenom, salaire, salaire*12 FROM employes;

Utiliser un alias avec AS

fin row
  • SELECT nom, prenom, salaire, salaire*12 AS salaire_annuel FROM employes ORDER BY salaire_annuel DESC;
  • Le mot clef AS permet de définir un alias

Calculer une somme avec SUM, et une moyenne avec AVG

  • SELECT SUM(salaire*12) FROM employes;
  • AVG faire la moyenne (average)SELECT AVG(salaire*12) FROM employes; ou la moyenne mensuelle : SELECT AVG(salaire) FROM employes;

Arrondir avec ROUND

  • Arrondir un calcul : SELECT ROUND(AVG(salaire)) FROM employes;
  • Faire un alias SELECT ROUND (AVG(salaire)) AS moyenne_mensuelle FROM employes;

Compter avec COUNT

  • Compter le nombre d'employées : SELECT COUNT(*) FROM employes WHERE sexe='f';
  • Compter le nombre d'employés : SELECT COUNT(*) FROM employes WHERE sexe='m';

Isoler une valeur minimum / maximum avec MIN ou MAX

  • Trouver le salaire minimum SELECT MIN(salaire) FROM employes;
  • Par contre cette requête ne fonctionnera pas SELECT prenom, MIN (salaire) FROM employes; Pour trouver le salaire le plus bas et la personne qui gagne ce salaire il faut écrire SELECT prenom, salaire FROM employes WHERE salaire = (SELECT MIN(salaire) FROM employes);le système va d'abord excécuter la requête entre parenthèse pour connaitre le résultat et l'utiliser comme source pour l'autre requête
  • ....
  • Trouver le salaire maximum SELECT MAX(salaire) FROM employes;
  • Trouver qui gagne le plus SELECT prenom, salaire FROM employes WHERE salaire = (SELECT MAX(salaire) FROM employes ); On appel cela une requête imbriquée
  • Trouver qui gagne 1390 Euros SELECT nom, prenom, salaire FROM employes WHERE salaire = 1390;
  • Pour trouver le salaire horaire SELECT nom, prenom, salaire, salaire/150 AS salaire_heure FROM employes ORDER BY salaire_heure ASC;

Condition IN avec plusieurs valeurs

  • Trouver les employes qui travaillent à la comptabilité et au service informatique : SELECT nom, prenom, service FROM employes WHERE service IN ('comptabilité', 'informatique');
  • ...
  • Le = (égal) permet d'annoncer une seule valeur
  • IN permet d'annoncer plusieurs valeurs
  • ...
  • Trouver les employes qui ne sont pas au service informatique et à la comptabilité avec NOT IN SELECT nom, prenom, service FROM employes WHERE service NOT IN ('comptabilité', 'informatique') ORDER BY service;

Conditions multiples, différents champs et valeurs

  • Les employés travaillent au service commercial et qui gagnent un salaire inférieur ou églal à 2000 Euros SELECT nom, prenom, salaire, service FROM employes WHERE service = 'commercial' AND salaire <= 2000;

ordre de priorité sur les conditions AND et OR

  • SELECT nom, prenom, service, salaire FROM employes WHERE service ='production' AND salaire = 1900 OR salaire = 2300;
  • Lorsqu'il y a AND er OR au sein de la même requête, le système lit la condition de la droite vers la gauche et il verra d'abord salaire= 1900 OR salaire = 2300 et ensuite service ='production'
  • Pour garantir l'ordre naturel de lecture de la gauche vers la droite on utilisera les parenthèses SELECT prenom, nom, service, salaire FROM employes WHERE service='production' AND (salaire= 1900 OR salaire=2300);

Les regroupements avec GROUP BY

  • Si on veut connaitre le nombre d'employés par service SELECT service, COUNT(*) AS nombre_employes FROM employes GROUP BY service;

Utilisation du mot-clef HAVING, ayant

  • On impose une condition COUNT et HAVING SELECT service, COUNT(*) AS nombre_employes FROM employes GROUP BY service HAVING COUNT(*) >2;

Les requêtes d'insertion avec INSERT

La requête INSERT INTO

  • Insérer un enregistrement : 1ère méthode INSERT INTO employes (prenom, nom, sexe, service, date_embauche, salaire) VALUES ('jean', 'richy', 'm', 'informatique', '2011-12-28', 1800);
  • Insérer un enregistrement : 2 ème méthode INSERT INTO employes VALUES (NULL, 'jean', 'richy', 'm', 'informatique', '2012-01-28', 1800); NULL nous permet de laisser le champ id_employes se remplir seul car il est auto-incrémenté

Les requêtes de modification

La requête UDPDATE

  • Augmenter le salaire d'un employé UPDATE employes SET salaire= 1871 WHERE id_employes = 990;
  • Il est recommandé de modifié un enregistrement en ciblant son id, celui-ci étant unique
  • Dans le cas ou plusieurs modifications s'opèrent UPDATE employes SET service= 'commercial', salaire = 1999 WHERE id_employes = 999;

La requête DELETE

  • Supprimer un enregistrement DELETE FROM employes WHERE id_employes = 388;
  • Supprimer plusieurs enregistrements DELETE FROM employes WHERE service= 'informatique';
